## Project Background

我是一个非职业程序员，主要依靠 AI 辅助开发来完成产品搭建和长期维护。我真正需要的不是一次性生成很多代码，而是一套清晰、稳定、可长期约束 AI 的后端 API 底座。

这个项目的目标是沉淀一个可复用的 NestJS 后端模板，用于后续开发公益救援队管理系统、内部管理系统、小程序 API、AI 应用后台，以及其他中小型业务系统。

项目优先级是：稳定、清晰、可维护、AI 友好，而不是功能堆叠。所有设计都应避免过度工程化，优先采用简单、显式、强约束的实现方式。v1 阶段只做最小闭环：NestJS + Prisma + PostgreSQL + Docker Compose + Swagger + JWT 登录 + 用户 CRUD + 简单角色权限 + 统一异常与返回格式。

这个底座的核心价值不是代码本身，而是规范文档、目录约束、接口规范、数据库规范、错误码规范、安全规范和 AI 开发规则。任何 AI 在修改本项目时，都必须先阅读并遵守项目规则，不得擅自引入超出当前阶段范围的复杂能力，例如 Redis、BullMQ、refresh token、RBAC 权限表、组织架构、多租户、文件上传实现、AI provider、pgvector、GraphQL 或微服务。

后续新增业务时，应在既有规范内平铺添加模块，保持模块边界清楚、接口风格统一、数据库模型可解释、权限逻辑简单明确。项目追求的是可持续二开，而不是短期炫技。
